Обычная функция - `this` определяется в момент вызова.
Стрелочная функция - берет внешний `this`.
Вызов без объекта - `this === undefined`.

## Конструктор

Создаёт объект оператор `new`.

`return` у Конструктора:

- конструктор явно возвращает объект → результатом становится этот объект;
- конструктор возвращает примитив или ничего → результатом становится this.

## Опциональная цепочка

Синтаксис *опциональной цепочки (optional chaining)* имеет 3 формы:

1. `obj?.prop` – возвращает `obj.prop` если `obj` существует, в противном случае (если `obj === undefined/null`) `undefined`.
2. `obj?.[prop]` – возвращает `obj[prop]` если `obj` существует, в противном случае `undefined`.
3. `obj.method?.()` – вызывает `obj.method()`, если `obj.method` существует, в противном случае возвращает `undefined`.

## Преобразование объектов в примитивы

Операторы не могут напрямую работать с объектами. Объекты преобразуется в примитивы -> операция -> примитив как ответ.
Программа смотрит на *hint*, чтобы преобразовать объект в один из 3 вариантов:

1. `string`. Когда ожидается строка. Например: `alert`;
2. `number`. В случае мат. операций.
3. `default`. Когда оператор не уверен какой тип ожидать.

Для преобразования, JS ищет 3 метода:

1. `obj[Symbol.toPrimitive](hint)` — метод с ключом `Symbol.toPrimitive`.

    ```javascript
    obj[Symbol.toPrimitive] = function (hint) {

    }   
    ```

2. Если `hint === "string"`, JavaScript пробует `obj.toString()`, затем `obj.valueOf()`.
3. Если `hint === "number"` или `hint === "default"`, JavaScript пробует `obj.valueOf()`, затем `obj.toString()`.

Обычный объект возвращает:

- `[object Object]` для `toString`.
- `object` для `valueOf`.

`toString` и `valueOf` проигнорируют, если преобразование возвращает объект. Но выйдет ошибка, только если все способы (`toString`, `toValue`) выдали объект.
`System.toPrimitive(hint)` выдаст сразу ошибку при возврате объекта.
